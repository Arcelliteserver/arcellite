A lot happened this month. New AI providers, K8s updates, much more data on your usage page. Oh, and self-learning coding agents and a website that finally looks like the product behind it. Let's tuck in.

ngrok.ai now supports the Anthropic SDK for native access to Claude APIs. Ditch the compatibility layer and use Anthropic's /v1/messages API to use Claude-only features like prompt caching, thinking parameters, and beta headers. Just change the base_url and you're set.

Start using the Anthropic SDK ->

We've also added support to Groq, Hyperbolic, InceptionLabs, and Inference.net to ngrok.ai. Pick a model from the model catalog, point your SDK at your gateway's URL, and let us handle routing, failover, and observability.

v0.22 of the ngrok Kubernetes Operator lets you install multiple instances in the same cluster. Why? Multi-tenancy. Team A gets their own operator and their own Traffic Policy configs to make sure changes in their namespace don't ripple over to team B, C, and beyond.2

See other examples and bugfixes ->

Your usage page got some big love. Many of you have asked for more details, and now you can now see remaining usage credits, which endpoints are incurring billable hours, transfer tallies, and Traffic Policy usage. Friendly note: You'll only see this if you're on our latest plans. Not sure? Check out your billing page.

Check out the new usage page ->

ngrok's Windows binaries are now signed with an EV certificate. The new DigiCert Extended Validation certificate establishes immediate trust with Windows, which means fewer (ideally zero) scary popups on fresh downloads. You don't have to do anything, your next installation or update will use this new certificate automatically.

Read about the change ->

There's a use case page for the why, a quickstart for the impatient, a full tutorial for the thorough, and then about a dozen more guides for security, reliability, and production-readiness. Sorry not sorry.
